# 📊 배당판 오버레이 — 실제 배당판 실측 체크리스트

> 목적: 오버레이(overlay.js)가 **기존 배당 수집 기능에 전혀 영향을 주지 않음**을 실제 배당판에서 최종 확인한다.
> 오버레이는 **읽기 전용·완전 격리**로 설계됐고 합성 DOM 격리 테스트는 통과했으나, 실제 사이트에서 1회 실측으로 완결한다.

---

## 사전 준비
- 서버 실행 중(`http://127.0.0.1:8011`) — 분석기 탭 1개 열어두기
- 확장 버전 **v2.1.19** 이상 (`chrome://extensions` 에서 확인)

---

## ✅ 단계별 체크리스트

### 1. 확장 재설치 (코드 반영)
- [ ] `chrome://extensions` 접속 → 개발자 모드 ON
- [ ] 기존 "경마 배당 자동수집기" **제거** 또는 **새로고침(↻)**
  - ZIP 설치 사용 시: `chrome-extension.zip` 압축 해제 후 **"압축해제된 확장 프로그램을 로드"** 로 폴더 선택
- [ ] 버전이 **2.1.19** 로 표시되는지 확인
- [ ] 오류(빨간 Errors) 없는지 확인

### 2. 배당판 열기
- [ ] 실제 배당판 페이지 열기 (keiba.go.jp 또는 사설 배당판 asyukk34)
- [ ] 우측 상단에 작은 **`📊 분석` 칩**이 보이는지 확인 (오버레이 기본 OFF 상태)
- [ ] 팝업 열어서 raceKey/발주시각 자동 감지 정상인지 확인

### 3. 오버레이 ON (수집 켠 상태에서)
- [ ] 팝업에서 **자동 전송 ON** (기존처럼 수집 시작)
- [ ] 오버레이 켜기 (둘 중 편한 방법):
  - 팝업의 **"📊 배당판 오버레이" 스위치 ON**, 또는
  - 배당판 우측 상단 **`📊 분석` 칩 클릭**
- [ ] 분석 패널이 나타나고 **마감 카운트다운 / 유력마 / 급락 / 추천**이 표시되는지 확인
- [ ] 패널 **헤더를 드래그**해 위치를 옮기고, 페이지 새로고침 후 **같은 위치로 복원**되는지 확인

### 4. 수집 정상 확인 (오버레이 ON 상태) — ⭐핵심
- [ ] 분석기 탭에서 **📥 확장 수집 3종(복승/쌍승 매트릭스)** 이 정상 갱신되는지
- [ ] 배당판 배지(확장 아이콘)에 수집 카운트(초록 숫자)가 뜨는지
- [ ] 팝업 **"3종 수집: N"** 정상 증가 확인
- [ ] 콘솔(F12)에 수집 관련 **에러 없는지** 확인
  - 참고: 오버레이는 `<div>`만 주입하므로 `document.querySelectorAll('table')` 수집에 영향 없음

### 5. 오버레이 OFF
- [ ] 팝업 스위치 OFF 또는 패널 **✕** 클릭 → 패널 사라지고 **`📊 분석` 칩만** 남는지
- [ ] (완전 제거 원하면) 콘솔에서 `chrome.storage.local.set({overlayKill:true})` → 칩까지 제거되는지

### 6. 수집 정상 확인 (오버레이 OFF 상태) — ⭐핵심
- [ ] 오버레이를 끈 뒤에도 **3종 수집 카운트가 계속 정상 증가**하는지
- [ ] 즉시 분석(🚨)·결과 자동수집 등 **기존 기능 전부 정상**인지

---

## 🔴 문제 발생 시 즉시 롤백
| 상황 | 조치 |
|---|---|
| 오버레이가 화면을 가림 | 패널 **✕** 또는 팝업 스위치 OFF |
| 오버레이 완전 제거(칩 포함) | 콘솔: `chrome.storage.local.set({overlayKill:true})` |
| 수집이 이상함(오버레이 의심) | 위 kill 실행 후 재확인 — 그래도 이상하면 오버레이 무관 |
| 확장 자체 롤백 | `chrome://extensions` 에서 이전 버전 zip 재설치 |

> ⚠️ 오버레이는 `overlayKill` 한 줄로 **DOM에서 완전히 사라지며**, 수집 엔진(content.js)은 **바이트 단위로 변경되지 않았으므로**(git diff 0) 오버레이를 꺼도/제거해도 수집은 항상 동일하게 동작한다.
