# 경마 배당 자동수집기 (Chrome 확장 프로그램)

`keiba.go.jp` 배당판에서 **단승/복승 배당을 자동으로 읽어** 경마분석서버
(`http://127.0.0.1:8011`)의 `POST /api/odds/snapshot` 으로 전송합니다.

## 파일 구조

```
chrome-extension/
├─ manifest.json     # 확장 프로그램 정의 (MV3)
├─ content.js        # 배당 추출: 마번·말이름·단승·복승 매트릭스 → JSON
├─ background.js     # 서버 전송 담당 (mixed-content/CORS 우회)
├─ popup.html        # 확장 아이콘 팝업 UI
├─ popup.js          # 팝업 로직 (상태·간격·수동전송)
├─ icons/            # 아이콘 16/48/128
└─ README.md         # 이 문서
```

## 설치 (개발자 모드 → 압축해제된 폴더 로드)

1. Chrome 주소창에 `chrome://extensions` 입력 후 이동
2. 우측 상단 **개발자 모드(Developer mode)** 토글을 **켬**
3. **압축해제된 확장 프로그램을 로드(Load unpacked)** 클릭
4. 이 `chrome-extension` 폴더를 선택
5. 툴바에 🐎 아이콘이 나타나면 완료

> 코드 수정 후에는 `chrome://extensions` 에서 이 확장의 **새로고침(↻)** 을 눌러 반영합니다.

## 사용법

1. 경마분석서버를 먼저 실행합니다 (`python app.py` → `http://127.0.0.1:8011`).
2. `keiba.go.jp` 의 **배당 페이지**(単勝・複勝 또는 馬連)를 엽니다.
3. 🐎 아이콘을 클릭하면 팝업이 열립니다.
   - **서버 연결 ●** : 초록이면 8011 서버 연결됨 / 빨강이면 서버 꺼짐
   - **raceKey** : 페이지에서 자동 감지(회장·경주번호). 비면 직접 입력
   - **자동 전송** : 켜면 선택한 간격(15초~5분)마다 자동 전송
   - **지금 전송** : 현재 화면의 배당을 즉시 1회 전송
   - **마지막 전송** : 성공/실패 시각과 누적 스냅샷 수 표시

## 결과(1~3착) 자동 수집 (v1.1)

keiba 결과 페이지(`RaceMarkTable` = 레이스 성적표)를 열면 **자동으로 1~3착을 추출해
`POST /api/results/auto` 로 전송**합니다. 팝업의 `🏁 결과(1~3착) 전송` 버튼으로 수동 전송도 가능.

- 감지: URL에 `result`/`RaceMarkTable` 포함 또는 `着順` 헤더가 있는 표
- 추출: `着順 | 枠 | 馬番 | 馬名` 표에서 착순 숫자가 있는 행만 (失格/中止/除外 제외)
- 전송 형식: `{ raceKey, results:[{rank,no,name}], source }` → 서버 `{ ok, saved, top3 }`
- 저장: 서버 `results_store.json` (raceKey별 착순 + top3), 조회 `POST /api/results/get`

## 경주 타이머 (v1.2 · 모든 탭 상단 고정)

`timer.js`가 **모든 웹페이지 상단**에 카운트다운 바를 띄웁니다 (keiba·유튜브 등 어디든).

- **발주시각(HH:MM) 입력** → 카운트다운. `chrome.storage`로 **모든 탭이 같은 타이머** 공유
- **남은 시간 크게 표시** + 상태(🟢 베팅가능 / 🔴 마감), 5분 이하 노랑·1분 이하 빨강
- **단계 알림**(🔔 소리 + 문구): 10분전 `배당판 전송 시작하세요` / 5분전 `배당 추세 확인` /
  3분전 `빠른 비교 실행` / 1분전 `베팅 마감 임박!`
  - 소리는 **포커스된 탭에서만**(여러 탭 동시 비프 방지), 문구는 모든 탭 표시
- `✕`로 해당 탭에서 숨김(새로고침 시 재표시). `<all_urls>` 주입이라 설치 시 "모든 사이트" 권한이 필요합니다.

## 전체 자동 수집 — 3종 원버튼 (v1.3)

keiba **경주 오즈 페이지**에서 팝업의 `⚡ 전체 자동 수집` 한 번으로:
1. 복승(馬連 `OddsUmLenFuku`) 2. 쌍승(馬単 `OddsUmLenTan`) 3. 삼복승(3連複 `Odds3LenFuku`)
세 페이지를 **순서대로 동일출처 fetch → 파싱 → 서버로 한 번에 전송**합니다.

> **왜 탭을 클릭하지 않나?** keiba에서 복승/쌍승/삼복승은 **별도 페이지**(URL이 다름)라,
> 탭(링크)을 클릭하면 페이지가 통째로 이동해 확장 스크립트가 리셋됩니다. 그래서 페이지 이동 없이
> 세 오즈 URL을 순차 `fetch` 하여 **"복승→쌍승→삼복승 순서대로 클릭"과 동일한 결과**를 안전하게 얻습니다.

- 전송: `POST /api/odds/triple/ingest` `{raceKey, quinella[], exacta[], trio[]}` → `{ok, counts}`
- 서버 앱(배당판 캡처 탭)의 `📥 확장 수집 3종 불러오기` → `GET /api/odds/triple/latest` 로 받아
  **복승 삼각·쌍승 정방 매트릭스 + 삼복승 상위** 표시 (13두 기준 78·156·286 검증)

### 진행상황 표시 · 자동간격 3종 수집 (v1.3.2)

- **진행상황 실시간 표시**: 수집 중 팝업에 `복승 수집중… → 쌍승 수집중… → 삼복승 수집중… → 서버 전송중…`
  → 완료 시 `3종 수집 완료 ✅ 복승 78·쌍승 156·삼복승 286` (content.js가 `storage.tripleProgress` 로 브로드캐스트)
- **자동 대상 선택**: 팝업 `자동 대상` = `전체 3종 수집`(기본) / `단승 스냅샷`.
  `자동 전송`을 켜면 선택한 간격마다 **전체 3종 수집이 자동 실행**됩니다 (예: 2분마다 3종 갱신).
- **페이지 가드**: keiba 오즈 페이지(`k_raceDate`·`k_raceNo` URL)가 아니면
  `배당판 페이지가 맞는지 확인하세요` 안내 후 중단.

## 사설 배당판 + 범용 파서 (v1.4.0)

keiba.go.jp 외에 **사설 배당판**(예: `ks1.dke-d11diw.site`, 클래스 `odds_table`/`odds_content`)도
지원합니다. URL 로 사이트를 자동 감지해 파서를 분기합니다.

- **[사이트 자동 감지]** `detectSite()` — `keiba.go.jp`→keiba 파서 / `asyukk`·`qwqwd`→범용 매트릭스 파서(`odds_content`) / 기타→범용(소수점 숫자 셀)
- **[범용 매트릭스 파서]** `parseMatrixTable()` — 헤더 행에서 **열 마번 축**(cellIndex→마번), 각 행 첫 정수 셀 = **행 마번**,
  배당 셀(`.odds_content` 또는 소수 숫자)만 추출. `-`·빈칸·1.0 미만은 자동 무시.
  열 마번 축이 2개↑면 **매트릭스(복승 조합)**, 아니면 **리스트(단승)** 로 해석.
- **[검증]** `validateExtraction()` — 마번 1~16 · 배당 ≥1.0 · 최소 3조합. 실패 시 미리보기에 ⚠ 경고.
- **[미리보기]** 팝업 `🔍 미리보기 (전송 전 확인)` → 사이트·raceKey·단승 두수·복승 조합수 + **상위 10조합(배당 낮은순)** +
  검증 경고 표시 → `✅ 확인 후 전송`.
- **[전송 경로]** 사설/범용 사이트는 **복승 매트릭스 → `POST /api/odds/triple/ingest`(quinella)** 로 보내
  서버 앱의 매트릭스 UI 에 바로 표시되게 하고, **단승이 있으면 `POST /api/odds/snapshot` 도 함께** 전송합니다.
  (snapshot 은 단승이 비면 거부하고 매트릭스를 저장하지 않기 때문)

> **raceKey**: 사설 사이트는 URL 에 keiba 파라미터가 없어 자동 감지가 안 될 수 있습니다.
> 팝업 `raceKey` 칸에 직접 입력하세요(예: `2026-07-02 서울 5R`).
>
> **미러 도메인 교체 시**: 사설 사이트 도메인(`*.dke-d11diw.site`)이 바뀌면 `manifest.json` 의
> `content_scripts.matches` 와 `host_permissions` 에 새 도메인을 한 줄 추가하고 확장을 새로고침하세요.

## 전송 데이터 형식

서버 계약(`/api/odds/snapshot`)에 맞춰 **단승 배당 맵**을 보냅니다.
복승 매트릭스·말이름은 확장 필드로 함께 실어 보내며, 서버가 무시해도 안전합니다.

```json
{
  "raceKey": "2026-07-02 大井 12R",
  "odds": { "1": 3.5, "2": 2.1, "3": 4.5 },
  "horses": [
    { "no": 1, "name": "…", "win": 3.5, "place": { "min": 1.2, "max": 1.6 } }
  ],
  "quinella": {
    "pairs": [ { "a": 1, "b": 2, "odds": 8.4 } ],
    "matrix": { "1": { "2": 8.4 }, "2": { "1": 8.4 } }
  },
  "capturedAt": "2026-07-02T05:00:00.000Z",
  "source": "https://www.keiba.go.jp/…"
}
```

## 배당 추출 방식 / 튜닝

실제 keiba.go.jp 배당 페이지 DOM 을 확인해 아래 구조에 맞춰 파싱합니다.
(헤더 라벨로 컬럼을 찾으므로 열 순서가 바뀌어도 견고합니다.)

- **단승·복승 페이지** `OddsTanFuku` — `table.odd_popular_table_02`
  헤더: `枠 | 馬番 | 馬名 | 単勝オッズ | 複勝オッズ(3着払い) | …`
  → `extractHorses()` 가 `馬番`(枠 아님) 기준으로 마번·말이름·단승·복승을 읽음.
  복승은 `"4.3-"` + `"13.6"` 두 셀로 나뉘어 있어 범위로 합칩니다.
- **복승(馬連) 매트릭스** `OddsUmLenFuku` — `table.odd_ranking_table`
  행: `組合せ("3-8") | オッズ("3.8") | 人気` → `extractQuinella()` 가 전 조합 파싱.
- `extractRaceKey()` : URL 파라미터(`k_raceDate`/`k_babaCode`/`k_raceNo`) 우선,
  본문 헤더(`2026年7月1日 大井 第11競走`)로 보강 → 예: `2026-07-01 大井 11R`.

> 실측 검증: 大井 11R(13두)에서 단승 13개·복승 78조합(=₁₃C₂) 정상 추출 확인.

배당 발매 전이면 keiba 페이지 자체가 "オッズの情報がありません"를 반환하므로,
**발매 시간대의 배당 페이지**에서 실행해야 합니다. 그 외 추출이 비면
"단승 배당을 찾지 못했습니다"가 표시됩니다.

## 문제 해결

| 증상 | 원인 / 해결 |
|------|-------------|
| 서버 ● 빨강 | `app.py` 미실행. `python app.py` 로 8011 서버 켜기 |
| "페이지 응답 없음" | 확장 로드 후 keiba 페이지를 **새로고침** |

| 배당 0개 | 배당 페이지(単勝複勝/馬連)가 맞는지 확인, 발매 시간대인지 확인 |
