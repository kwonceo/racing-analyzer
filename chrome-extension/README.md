# 경마 배당 자동수집기 (Chrome 확장 프로그램)

`keiba.go.jp` 배당판에서 **단승/복승 배당을 자동으로 읽어** 경마분석서버
(`http://127.0.0.1:8011`)의 `POST /api/odds/snapshot` 으로 전송합니다.

## 파일 구조

```
chrome-extension/
├─ manifest.json     # 확장 프로그램 정의 (MV3)
├─ content.js        # 배당 추출: 마번·말이름·단승·복승 매트릭스 → JSON
├─ background.js     # 서버 전송 담당 (mixed-content/CORS 우회)
├─ popup.html        # 확장 아이콘 팝업 UI
├─ popup.js          # 팝업 로직 (상태·간격·수동전송)
├─ icons/            # 아이콘 16/48/128
└─ README.md         # 이 문서
```

## 설치 (개발자 모드 → 압축해제된 폴더 로드)

1. Chrome 주소창에 `chrome://extensions` 입력 후 이동
2. 우측 상단 **개발자 모드(Developer mode)** 토글을 **켬**
3. **압축해제된 확장 프로그램을 로드(Load unpacked)** 클릭
4. 이 `chrome-extension` 폴더를 선택
5. 툴바에 🐎 아이콘이 나타나면 완료

> 코드 수정 후에는 `chrome://extensions` 에서 이 확장의 **새로고침(↻)** 을 눌러 반영합니다.

## 사용법

1. 경마분석서버를 먼저 실행합니다 (`python app.py` → `http://127.0.0.1:8011`).
2. `keiba.go.jp` 의 **배당 페이지**(単勝・複勝 또는 馬連)를 엽니다.
3. 🐎 아이콘을 클릭하면 팝업이 열립니다.
   - **서버 연결 ●** : 초록이면 8011 서버 연결됨 / 빨강이면 서버 꺼짐
   - **raceKey** : 페이지에서 자동 감지(회장·경주번호). 비면 직접 입력
   - **자동 전송** : 켜면 선택한 간격(15초~5분)마다 자동 전송
   - **지금 전송** : 현재 화면의 배당을 즉시 1회 전송
   - **마지막 전송** : 성공/실패 시각과 누적 스냅샷 수 표시

## 전송 데이터 형식

서버 계약(`/api/odds/snapshot`)에 맞춰 **단승 배당 맵**을 보냅니다.
복승 매트릭스·말이름은 확장 필드로 함께 실어 보내며, 서버가 무시해도 안전합니다.

```json
{
  "raceKey": "2026-07-02 大井 12R",
  "odds": { "1": 3.5, "2": 2.1, "3": 4.5 },
  "horses": [
    { "no": 1, "name": "…", "win": 3.5, "place": { "min": 1.2, "max": 1.6 } }
  ],
  "quinella": {
    "pairs": [ { "a": 1, "b": 2, "odds": 8.4 } ],
    "matrix": { "1": { "2": 8.4 }, "2": { "1": 8.4 } }
  },
  "capturedAt": "2026-07-02T05:00:00.000Z",
  "source": "https://www.keiba.go.jp/…"
}
```

## 배당 추출 방식 / 튜닝

실제 keiba.go.jp 배당 페이지 DOM 을 확인해 아래 구조에 맞춰 파싱합니다.
(헤더 라벨로 컬럼을 찾으므로 열 순서가 바뀌어도 견고합니다.)

- **단승·복승 페이지** `OddsTanFuku` — `table.odd_popular_table_02`
  헤더: `枠 | 馬番 | 馬名 | 単勝オッズ | 複勝オッズ(3着払い) | …`
  → `extractHorses()` 가 `馬番`(枠 아님) 기준으로 마번·말이름·단승·복승을 읽음.
  복승은 `"4.3-"` + `"13.6"` 두 셀로 나뉘어 있어 범위로 합칩니다.
- **복승(馬連) 매트릭스** `OddsUmLenFuku` — `table.odd_ranking_table`
  행: `組合せ("3-8") | オッズ("3.8") | 人気` → `extractQuinella()` 가 전 조합 파싱.
- `extractRaceKey()` : URL 파라미터(`k_raceDate`/`k_babaCode`/`k_raceNo`) 우선,
  본문 헤더(`2026年7月1日 大井 第11競走`)로 보강 → 예: `2026-07-01 大井 11R`.

> 실측 검증: 大井 11R(13두)에서 단승 13개·복승 78조합(=₁₃C₂) 정상 추출 확인.

배당 발매 전이면 keiba 페이지 자체가 "オッズの情報がありません"를 반환하므로,
**발매 시간대의 배당 페이지**에서 실행해야 합니다. 그 외 추출이 비면
"단승 배당을 찾지 못했습니다"가 표시됩니다.

## 문제 해결

| 증상 | 원인 / 해결 |
|------|-------------|
| 서버 ● 빨강 | `app.py` 미실행. `python app.py` 로 8011 서버 켜기 |
| "페이지 응답 없음" | 확장 로드 후 keiba 페이지를 **새로고침** |
| "raceKey 를 만들 수 없습니다" | 팝업의 raceKey 칸에 직접 입력 (예: `大井 12R`) |
| 배당 0개 | 배당 페이지(単勝複勝/馬連)가 맞는지 확인, 발매 시간대인지 확인 |
