## v2.3.0 (2026-07-05) — 이상감지 영구 누적 + 마감 전 3단계 알림 · **현재 최신**

### 추가된 기능
- **일본경마 삼복승 제거 — 복승+쌍승만 사용**(확장 v2.1.9): ①**수집 제거** — `collectTripleKeiba`(keiba fetch)·`collectTripleByTabs`(사설 탭클릭) 양 경로에서 삼복승(3連複) fetch/탭클릭 완전 제거(일본=복승+쌍승, 한국=복승). 기존 삼복승 수집 함수는 코드로 보존(호출만 안 함). ②**추천 제거** — 삼복승 배당 미수집 시 `_triple_analyze`가 삼복승 추천(메인/보험/near-miss/대규모급락 보험)을 전체 제외하고 복승 메인/보조·복승 신호로 재배분(합≈100). 삼복승 로직 자체는 보존(배당 있으면 그대로 동작). 쌍승 역전은 참고로 계속 표시. ③**UI 정리** — 팝업 라벨·수집 상태·index 안내 문구를 복승·쌍승 중심으로 정리
- **신호말 전체 복승 조합 추천(고배당 놓침 방지)**(`_signal_combo_bets`): 이상감지 신호가 있는 말(집중급락·복승/단승 급락·쌍승역전·이상감지말)들의 **모든 복승 조합을 추천에 추가**(이미 추천된 조합 제외, 배당 있는 조합만, 고배당 우선). 배당대별 tier 표기(고배당신호 50배+·신호기반·낮은배당신호). `복승 신호` 라벨로 betRecommend 포함 → 결과 입력 시 적중 판정·학습에 반영(예: 코치 3경주 결과 2-6-7에서 2/6 신호 있던 **2+6 복승 147배**를 추천에 포함해 적중 인정). 마감 후에는 미추가. 프론트 베팅표 예상배당 셀에 tier 배지
- **결과 입력 4착 확장 + 삼복승 near-miss 학습**: ①**4착 입력** — 일본경마 복기·결과기록·history 결과 입력 폼에 4착 필드 추가(적중 판정은 복승 1+2·삼복승 1+2+3 기존 유지). ②**아깝게 4착(거의 적중)** — 추천 삼복승 2두가 top3 + 나머지 1두가 정확히 4착이면 `near_miss`/`near_miss_horse`/`trio_near_miss` 기록(`_apply_result_learning`), 복기 리포트에 "🟡 아깝게 4착 - 거의 적중" 표시. ③**4착 빈번 말 삼복승 보험픽 우선** — `data/near_miss.json` 누적, 2회+ 4착 말이 이번 출전마에 있으면 `_triple_analyze`가 `삼복승 보험(4착빈번)` 픽 자동 추가(마감 전만). 통계 탭 `🟡 삼복승 아깝게 4착` 카드 + `GET /api/learning/near-miss`

### 수정된 버그
- **경주 전환 시 배당 데이터 잔존 오검출 수정**: 새 경주로 넘어가도 이전 경주 배당이 남아 변동폭이 비정상적으로 크게 계산되던 문제(예: 이전 5+6=147.4 → 새 경주 5+6=5.7 → -96.1% 오검출) 해결. ①**경주 전환 감지**(`_baseline_reset_needed`) — 직전 대비 공통 복승 조합의 60%+가 90%+ 급락하면 다른 경주 잔존으로 판단, `triple_ingest`가 이전 history 완전 초기화 + 이번 수집을 새 기준값으로(`baselineReset`). ②**첫 수집 기준값** — 새 경주 첫 수집은 변동 계산 없이 기준값으로만 저장·"🎯 기준값 설정됨" 표시(`baselineSet`), 둘째 수집부터 변동 계산. ③**비정상 변동폭 필터** — 95%+ 급락 개별 조합은 복승/단승 drops에서 제외, 다수 발생 시 자동 재설정 + "⚠️ 비정상 변동폭 - 기준값 재설정" 신호. `_history_append`도 재설정 시 이상감지 생략·스냅샷 `baseline_reset` 표기. 서버단 방어라 확장 raceKey 갱신 지연/재사용에도 안전
- **현재 경주 자동 감지 오류 수정(확장 v2.1.7)**: 사설 보드에서 화면은 `고쿠라 4경주`인데 raceKey가 `모리오카 2경주`로 잘못 잡히던 문제. `extractRaceKey`가 경마장 목록(고쿠라 미포함)에서 본문의 첫 매칭(모리오카)을 집던 것을, **하단 네비 "이전 \<경마장\> \<N\>경주 … 다음"에 앵커링해 현재 경주를 직접 추출**하도록 수정(목록에 없는 경마장도 잡힘). 폴백 목록에 JRA 중앙(고쿠라·삿포로·니가타·도쿄·나카야마·주쿄·교토·한신) 추가. 고쿠라/부산/니가타 시나리오 검증.
- **한국경마 쌍승/삼복승 수집 완전 차단(확장 v2.1.6)**: 부산 등 KRA 경주인데도 `market`=auto + raceKey 추출 실패(사설 보드 표기 다양) 시 삼복승 탭을 클릭하던 문제. `pageLooksKorean()` 추가 — raceKey가 비어도 **페이지 본문/URL에 KRA 경마장명(서울/부산/제주/과천) + 경마 맥락**이 있으면 한국 확정. `isKoreaMode(raceKey, market)`로 판정 통일(종목=한국 OR raceKey OR 페이지). `collectTripleKeiba`·`collectTripleByTabs` 양 경로 적용 → 복승만 수집, 쌍승/삼복승 탭 클릭 완전 스킵. 로그 `[한국모드] 복승만 수집 - 쌍승/삼복승 생략`. 부산 화면 시나리오 포함 판정 검증.
