- **핵심 이상감지 수학 공식 5종**(`_win_exacta_reversal`/`_quinella_mismatch`/`_signal_confidence`): ①**쌍승 역전 감지 공식** — 단승(미수집 시 복승인기) 유력마 A vs 다른 말 B 방향 비교. 역전비율 = 쌍승(B→A)/쌍승(A→B). <0.95 🟡 역전 · <0.80 🔴 강한역전 · <0.60 🔴🔴 압도적 역전("🔄 역전감지: 단승 9번 유력이나 쌍승에서 2번 1착이 더 낮음 → 실질 1착 2번 가능성"). ②**복승 불일치 감지 공식** — 불일치점수 = 단승 예상 최저복승(1+2위 조합)/실제 최저복승. 1.2+ 🟡 · 1.5+ 🔴 · 2.0+ 🔴🔴("⚠️ 복승 불일치: 예상 A+B / 실제 A+C → C번 집중 자금 유입"). ③**초과급락 기준 수정** — 전체 평균 대비 10%+ 초과(또는 절대 10%+)는 무조건 집중신호(노이즈 아님, 앞 커밋 ABS_STRONG 연계). ④**종합 신뢰도 점수** — 초과급락40% + 쌍승역전35% + 복승불일치25% 합산(말별 0~100). 70+ 🔴 강력 · 40~69 🟡 참고 · 40 미만 노이즈. ⑤**추천 조합 자동 반영** — 신뢰도 강력(70+)·참고(40~69) 말을 신호조합 추천 최우선 순위로 반영(배당 높아도 신호 있으면 표시). 프론트 🎯 신호 품질 분석 카드에 종합신뢰도표·쌍승역전·복승불일치 추가. 격리 테스트(역전 0.5/불일치 3.0/신뢰도 합산/70+ 강력 트리거) 통과

- **쌍승 다중순위 역전 감지**(`_win_exacta_reversal` 확장 + `_reversal_level` 분리): 기존엔 **단승 1위 유력마 vs 나머지** 방향 역전만 감지했으나, 이제 **상위권(최대 4위) 다른 순위쌍(2·3·4위 간)의 역전**도 함께 감지(예: 단승 2위가 실은 3위보다 아래로 시장이 판단하는 경우). ①**primary(1위 기준) 먼저·multiRank 뒤** — 항상 1위 기준 역전을 강한 순으로 앞에 두고 순위쌍 역전을 덧붙여, `wx[0]`(AI 피처 `reversal_ratio`)·`[:3]`(역배열 배너)·`[:5]`(신호 리스트) 를 쓰는 기존 소비처 동작을 **완전 보존**. ②**노이즈 억제** — 하위 순위쌍은 노이즈가 크므로 **강한 역전(비율<0.80)만** 채택(1위 기준은 기존대로 <0.95). ③**신호 반영** — 각 역전의 challenger(실질 상위 지목마)가 `_signal_confidence` reversalScore·신호조합 추천(`_push_sig`)에 자동 반영. ④**프론트 구분 표시** — `🎯 신호 품질 분석` 카드의 쌍승 역전 목록에 `N·M위간` 보라 배지 + 헤더에 "상위권 순위쌍 역전 포함" 안내. `favRank`/`chalRank`/`multiRank` 필드 추가(기존 필드 불변). 격리 테스트(기존 primary 동작·문구 보존 · 2·3위 다중역전 감지 · 약한 하위쌍 제외 · wx[0] primary 우선순위 보존 · 방어 입력) 24항목 통과 + 기존 회귀(run_formula 23·run_flow 23) 통과
- **BMED 전략별 성과 학습**(`_recompute_learning_stats` `by_strategy`, record `bmed_strategy`/`inverse_detected`): 결과 입력마다 적용된 BMED 전략(보험형·압축형·역배열형·분산형·고배당도전형)별 적중률·수익 자동 집계 → 통계 탭 `📊 경마장별·월별·전략별 성과` 표에 `🎯 BMED 전략별` 추가. "어느 전략이 실제로 잘 맞는지" 데이터 축적(향후 전략 자동 가중의 근거). 격리 테스트(전략별 n/hit/rate/profit 집계) 통과

### 수정된 버그(계속)
- **자정 자동백업 무인 실행 hang 수정**(`scripts/backup_checkpoint.bat`·`register_daily_backup.bat`): 작업 스케줄러가 백업 배치를 무인 실행할 때 마지막 `pause`(키 입력 대기)로 **작업이 영원히 멈춰 자정 백업이 완료되지 않던** 문제. ①배치에 `/auto` 인자 모드 추가 → 무인 실행 시 `pause` 생략(파라미터 `%~1`로 판정, `endlocal` 후에도 유효). ②스케줄러 등록 시 `/TR "... /auto"`로 인자 전달. ③추가 방어 — `.git\index.lock` 감지 시(서버가 커밋 중) 이번 회차 스킵·로그 기록 후 다음 자정 재시도(git 잠금 충돌 방지).
- **자동전송 OFF 시 탭 클릭 방지**(확장 v2.1.12, `content.js` `AUTO_COLLECT` 핸들러): 사용자가 자동수집을 끈 직후 이미 발사된 in-flight 알람(stageT/fine)이 도착하면 복승/쌍승 탭을 강제 클릭해 **수동 베팅 화면을 방해**하던 문제. background 엔진에 autoSend 게이트가 있지만 끄는 순간 발사된 틱은 막지 못해, content.js `AUTO_COLLECT` 최종 진입부에 `autoSend` 재확인 가드 추가 — OFF면 `skipped:true`로 수집·탭클릭 모두 생략. 수동 수집(MANUAL_SEND)·결과 수집은 영향 없음.
- **한국경마 결과 매칭 버그 수정(형식 불일치)**(`_resolve_race_key`): 분석 로그 raceKey(`2026-07-05 서울 5경주`)와 결과 입력 raceKey(`서울 5`) 형식이 달라 "분석 경주 없음"이 뜨던 문제. ①**유연 매칭 리졸버** — 우선순위 ①완전 일치 ②경마장+경주번호 일치(오늘 날짜 후보 우선) ③경주번호만 일치(후보 유일할 때만). 날짜·"경주" 접미사 없어도 매칭. ②**`_area_num` 보강** — 날짜 먼저 제거(번호 오검출 방지) + `서울 5`(접미사 없음)도 번호 추출 + 한글·**일본어(가나·한자) 경마장명** 지원 + **부경=부산 별칭 정규화**(`_track_norm`). ③**결과학습 재구성** — triple_store(활성)가 churn으로 비어도 `_rec_from_history`가 odds_history 스냅샷에서 rec를 재구성해 분석 재현(결과 입력 정상 처리). ④**후보 소스 통합**(`_analyzed_race_keys`) — 활성 배당+odds_history+analysis_log 전체에서 매칭(한국·일본 중앙/지방 모든 경기). ⑤**일괄 등록도 유연화** — `_match_row_to_key`가 리졸버 사용(일본 포함). `record-result` 응답에 실제 매칭 key·`matchedFrom` 반환. 테스트: 서울 1~10 + 부산(부경) 1~7 + 오늘우선 + 완전일치 + 일본중앙 전부 매칭 + 재구성 결과학습 확인
- **새 경주 배당판 초반 미수집 → 가짜 급락·추천 불일치 수정**(긴급, 사가 6R·코치 7R): 새 경주 배당판을 초반에 못 끌어와 **opening/placeholder 배당**(복승 100배+, 예: 4+6=1585.1)이 캡처되면 ①실배당 정착 시 -80~-100% 가짜 급락이 이상감지 누적에 범람(사가 6R -94%/-98%/-100% 다수) ②추천 배당이 현재 배당판과 불일치(코치 7R "복승 2+9=7.1배"인데 실제 63.2배)하던 문제. **①opening 배당 정착 제외**(`_is_opening_settle`: 직전 100배+ & -80%↓ 급락은 기계적 정착 → 신호 아님) — `_history_append`(기록)·`_triple_analyze`(단승·복승 drops) 전 경로 적용. **②baseline 재설정 보강**(`_baseline_reset_needed`: 공통 조합 60%+가 opening 100배+에서 70%+ 정착 시에도 기준값 재설정 → 기록·표시 생략). **③이미 기록된 가짜 급락 숨김**(anomaly-feed에서 복승/단승 -88%↓ 필터). **④배당판 일치 검증**(`marketCheck`) — 추천 복승 메인이 배당판 최저 인기 조합보다 2.5배+ 비싸면(전적 편중/초반 미수집) `⚠️ 배당불일치` 경고 + **배당판 인기 조합을 `복승 인기(배당최저)`로 추천에 자동 추가**, 최저 복승도 80배+면 `⚠️ 배당불안정`(재수집 권장) 경고. 프론트 배너 표시. 격리 테스트(opening 정착 reset·개별 판정·정상경주 미오작동·불일치 감지+인기조합 추가) 통과
- **초과급락 신호말 노이즈 오판 → 추천 누락 수정**(긴급, 카나자와 10R 5-3-6): 시장 전체가 대규모 급락하면 3번(-17%)이 시장평균(예: -18%)에 묻혀 `excess≥0` → 노이즈 판정으로 추천에서 빠지던 문제(동일 패턴 3회째, 실제 2착 입상). **절대 급락 10%+(`avg≤-10`)는 시장평균과 무관하게 🔴 집중신호로 승격**(`_excess_drop_analysis` `ABS_STRONG=-10`) → `concentrated`에 포함되어 신호 조합 추천에 **반드시 반영**. 신호강도(`strength`=초과급락·절대급락 중 강한 쪽)로 점수(`_concentration_score`)·정렬·조합품질(`_combo_signal_quality`) 일관 계산. 집중신호 오판 사례는 `data/signal_lessons.json`에 학습 축적(`GET/POST /api/learning/signal-lessons`). 약한 급락(10% 미만)은 기존대로 노이즈 유지.
- **마감 오판 후 자동수집 영구 정지 수정**(확장 v2.1.11): 무변동 추정 마감에도 `autoSend:false`로 엔진을 완전히 끄고 `closed` 배너를 남겨, 한 번 오검출되면 수집이 꺼진 채 배너가 매 수집마다 재생성되던 문제. `_onRaceClosed`를 분기 — **확정 마감(DOM 발매마감)만 완전 정지**, **무변동(추정)은 `autoSend` 유지 소프트 일시중지 + 60초 뒤 재점검(`resumeCheck`) 자동 재개**(알림/배너 미표시). 배당 변동·새 경주 시 스스로 복구.
- **마감 오판(자동수집 조기 중단) 수정**(확장 v2.1.10): 발주 2분 40초 남았는데 "배당 5회 무변동"만으로 자동수집을 멈추던 문제. 확장이 발주시각을 못 잡으면(`deadline` 미설정) `!deadline`→무조건 무변동 마감을 적용하던 것을 제거. 배당판의 **실제 "남은시간"을 직접 파싱**(`pageRemainingMs`)해 **진짜 마감 임박(≤90초)일 때만** 무변동 마감 적용(발주시각 미검출 대비). 시간 정보 전무 시 DOM 마감 문구에만 의존.
- **첫 수집 가짜 급락 수정**: 배당판을 처음에 못 가져오거나(부분 로드) 시장 형성 초기 고배당일 때, 2번째 수집과 비교해 -90%대 가짜 급락이 이상감지 누적에 뜨던 문제. **첫 비교(수집 2건뿐)는 1틱 워밍업으로 급락 계산 보류**(`_triple_analyze` `market_forming`·`_history_append` 스냅샷 2건 이상부터 기록) → 2번째 수집을 기준으로 3번째부터 변동 계산.
- **경주 전환 시 배당 데이터 잔존 오검출 수정**: 새 경주로 넘어가도 이전 경주 배당이 남아 변동폭이 비정상적으로 크게 계산되던 문제(예: 이전 5+6=147.4 → 새 경주 5+6=5.7 → -96.1% 오검출) 해결. ①**경주 전환 감지**(`_baseline_reset_needed`) — 직전 대비 공통 복승 조합의 60%+가 90%+ 급락하면 다른 경주 잔존으로 판단, `triple_ingest`가 이전 history 완전 초기화 + 이번 수집을 새 기준값으로(`baselineReset`). ②**첫 수집 기준값** — 새 경주 첫 수집은 변동 계산 없이 기준값으로만 저장·"🎯 기준값 설정됨" 표시(`baselineSet`), 둘째 수집부터 변동 계산. ③**비정상 변동폭 필터** — 95%+ 급락 개별 조합은 복승/단승 drops에서 제외, 다수 발생 시 자동 재설정 + "⚠️ 비정상 변동폭 - 기준값 재설정" 신호. `_history_append`도 재설정 시 이상감지 생략·스냅샷 `baseline_reset` 표기. 서버단 방어라 확장 raceKey 갱신 지연/재사용에도 안전
- **현재 경주 자동 감지 오류 수정(확장 v2.1.7)**: 사설 보드에서 화면은 `고쿠라 4경주`인데 raceKey가 `모리오카 2경주`로 잘못 잡히던 문제. `extractRaceKey`가 경마장 목록(고쿠라 미포함)에서 본문의 첫 매칭(모리오카)을 집던 것을, **하단 네비 "이전 \<경마장\> \<N\>경주 … 다음"에 앵커링해 현재 경주를 직접 추출**하도록 수정(목록에 없는 경마장도 잡힘). 폴백 목록에 JRA 중앙(고쿠라·삿포로·니가타·도쿄·나카야마·주쿄·교토·한신) 추가. 고쿠라/부산/니가타 시나리오 검증.
- **한국경마 쌍승/삼복승 수집 완전 차단(확장 v2.1.6)**: 부산 등 KRA 경주인데도 `market`=auto + raceKey 추출 실패(사설 보드 표기 다양) 시 삼복승 탭을 클릭하던 문제. `pageLooksKorean()` 추가 — raceKey가 비어도 **페이지 본문/URL에 KRA 경마장명(서울/부산/제주/과천) + 경마 맥락**이 있으면 한국 확정. `isKoreaMode(raceKey, market)`로 판정 통일(종목=한국 OR raceKey OR 페이지). `collectTripleKeiba`·`collectTripleByTabs` 양 경로 적용 → 복승만 수집, 쌍승/삼복승 탭 클릭 완전 스킵. 로그 `[한국모드] 복승만 수집 - 쌍승/삼복승 생략`. 부산 화면 시나리오 포함 판정 검증.
- **(v2.1.5)** raceKey에 KRA 경마장명이 있으면 무조건 한국모드(`isKoreaByRaceKey`) — v2.1.6에서 페이지 감지까지 확장.

### 보완점
- ✅ **쌍승 역전 다중화 완료(공식·flip 양쪽)** — ①공식(`_win_exacta_reversal`)은 1위 기준 외 상위권 순위쌍(2·3·4위 간) 역전까지 감지. ②영구 flip 기록(`_history_append`)은 최저 1조합 → **상위 3개 저배당 조합** 각각의 방향 반전까지 누적(`EXA_REVERSAL_TOPN`). 남은 여지: 하위 조합/다단계(3개 초과) flip 누적은 노이즈 우려로 미채택(상수로 조정 가능).
