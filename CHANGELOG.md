## v2.3.0 (2026-07-05) — 이상감지 영구 누적 + 마감 전 3단계 알림 · **현재 최신**

### 추가된 기능
- **이상감지 결과 영구 누적**: 경주별 스냅샷(삭제 안 됨)에서 감지된 이상을 시간순·중복제거로 누적 표시 → 새 수집/마감 후에도 유지. `GET/POST /api/odds/anomaly-feed`(단승 급락·복승 급락·쌍승 역전, 심각도 🔴/🟡). 화면 좌하단 `🚨 이상감지 누적` 패널(3초 갱신)
- **쌍승 역전 영구 감지**: `_history_append`에 최저 쌍승 조합의 순서 반전(a,b)→(b,a) 감지 추가 → 스냅샷에 영구 기록
- **마감 전 3단계 알림**(`initClosingWatch`, `/api/auto/status` deadline 기반): T-1분30초 🔔소리2회+주황강조(현재 이상감지+추천) / T-1분 🚨소리3회+빨강강조(최종 메인 베팅+"지금 베팅하세요") / T-30초 ⏰소리4회+강한빨강("마지막 기회")
- **최종 알림에 누적 이상감지 요약 포함**: 각 단계 알림에 그동안 감지된 이상 전체 요약 + 메인 복승/삼복승 조합(`betRecommend`) 표기
- 격리 테스트(누적·중복제거·심각도·쌍승역전) + 라이브 수집→피드 파이프라인 검증

### 보완점
- 쌍승 역전은 "최저 쌍승 조합 방향 반전"만 감지(다중 순위 역전은 향후 확장 여지)
- 단계 알림은 확장이 `/api/auto/status`에 발주시각(deadline)을 올려야 작동(라이브 수집 시나리오) — 한국 PDF 전용(발주시각 미수집) 경주는 미발동

## v2.2.0 (2026-07-05) — 유력마/제거마 공식 고도화 + PDF 전경주 사전분석
태그 커밋 `ab772d7`

### 추가된 기능
- **유력마/제거마 선정 공식 전면 개선 [1~5]**: 유력마 점수(전적40+배당신뢰도30+기수20+적성10), 배당신뢰도(전적우수+배당낮음 +30 / 전적불량+배당낮음 -10 / 단독 0), 제거 점수(150배+ -40·80~149 -20·평균5착+ -30·기수<10% -10·급락 +30·쌍승 +20), verdict 컷 30/50/70, 수학적 확률(시장 1/배당×0.75·통합 0.6/0.4·기대값)
- **제거 공식 거리경험 -15 훅** 완결(`_elim_score(no_dist_exp)`, 데이터 확보 시 자동 활성)
- **PDF 전경주 사전분석**: 아침 1회 업로드 → 전경주 백그라운드 순차분석 → `data/prerace/` 경주별 영구저장 → `GET /api/korea/prerace`(목록)·`/api/korea/prerace/<key>`(즉시 로드)
- **사전분석 프론트 UI 배선**: 한국경마 탭에 `📅 저장된 사전분석` 패널 — 날짜별 그룹·경주 칩 클릭 시 세션과 무관하게 과거 분석 즉시 로드(`loadPreraceList`·`openPreraceRace`, 기존 렌더 흐름 재사용으로 푸터·결과입력까지 완전 동작)
- **실제 투자금액 손익 정확화**: 결과 입력에 `실수령 배당금(원)` 입력란 추가 — 입력 시 서버가 확정배당 추정 대신 `실수령 − 투자금`으로 정확 계산(`payout` 파라미터, 공란이면 기존 추정 유지). `record.payout_actual` 저장 + 격리 테스트 4케이스 통과
- **일괄 등록 경주별 손익 조정**: 요약 표를 경주별 `투자금·실수령` 편집 가능 표로 개편(라이브 손익 합계) + `POST /api/results/adjust`(가장 최근 학습 레코드 in-place 갱신, 중복 방지). 배당배수는 결과페이지 실제 확정배당 사용. `_recompute_pnl` 격리 6케이스 통과
- **사전분석 패널 자동 갱신**: `🔄 갱신` 버튼 + 아침 분석 진행 중 폴링에 맞춰 목록 실시간 갱신(`refreshPreraceIfOpen`)
- **거리 이력 수집 준비**(`fetch_kra.py`): 응답에 거리가 있으면 `rcDist`를 race+byHorse에 담도록 추가(다중 필드명 방어). 거리 보유 엔드포인트 확정이 활성 선행조건(현 `racedetailresult` 미반환·`API299_1` 500)
- 스펙 정합성 테스트 신설: `tests/run_formula.py`(23) · `tests/run_prerace.py`(18)

### 수정된 버그
