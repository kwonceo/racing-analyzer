## v2.2.0 (2026-07-05) — 유력마/제거마 공식 고도화 + PDF 전경주 사전분석 · **현재 최신**
태그 커밋 `ab772d7`

### 추가된 기능
- **유력마/제거마 선정 공식 전면 개선 [1~5]**: 유력마 점수(전적40+배당신뢰도30+기수20+적성10), 배당신뢰도(전적우수+배당낮음 +30 / 전적불량+배당낮음 -10 / 단독 0), 제거 점수(150배+ -40·80~149 -20·평균5착+ -30·기수<10% -10·급락 +30·쌍승 +20), verdict 컷 30/50/70, 수학적 확률(시장 1/배당×0.75·통합 0.6/0.4·기대값)
- **제거 공식 거리경험 -15 훅** 완결(`_elim_score(no_dist_exp)`, 데이터 확보 시 자동 활성)
- **PDF 전경주 사전분석**: 아침 1회 업로드 → 전경주 백그라운드 순차분석 → `data/prerace/` 경주별 영구저장 → `GET /api/korea/prerace`(목록)·`/api/korea/prerace/<key>`(즉시 로드)
- **사전분석 프론트 UI 배선**: 한국경마 탭에 `📅 저장된 사전분석` 패널 — 날짜별 그룹·경주 칩 클릭 시 세션과 무관하게 과거 분석 즉시 로드(`loadPreraceList`·`openPreraceRace`, 기존 렌더 흐름 재사용으로 푸터·결과입력까지 완전 동작)
- **실제 투자금액 손익 정확화**: 결과 입력에 `실수령 배당금(원)` 입력란 추가 — 입력 시 서버가 확정배당 추정 대신 `실수령 − 투자금`으로 정확 계산(`payout` 파라미터, 공란이면 기존 추정 유지). `record.payout_actual` 저장 + 격리 테스트 4케이스 통과
- 스펙 정합성 테스트 신설: `tests/run_formula.py`(23) · `tests/run_prerace.py`(18)

### 수정된 버그
- 중앙 JRA 결과 파싱 보강(전각숫자 ０-９·전각콜론·1着/2着/3着·複勝·三連複 컬럼)
- 지방 NAR 전적수집 안정화(`parseDebaTable` try/catch + `fetchDebaStarters` 2회 재시도)
- 합성 기수 제거 → KRA 실데이터 104명만 유지

### 보완점
- ✅ 프론트 `prerace` 목록 UI 배선 완료(한국탭 `📅 저장된 사전분석` 패널)
- ✅ 실제 투자금액 입력 필드 완료(실수령 배당금 → 정확 손익)
- `.gitignore` 정리: `.claude/` 제외 · `data/analysis_log/`는 학습 코퍼스로 추적 유지(주석 명시)

## v2.1.0 (2026-07-04 ~ 07-05) — KRA 실데이터 연동 + 학습 고도화
태그 커밋 `e5f9028`

### 추가된 기능
- **KRA 공공데이터 연동**(data.go.kr, `tools/fetch_kra.py`): 현직기수 104명 실제 복승률(`jockeys.json`), 경주성적 647경주(전적 3건+ 1,120두, `kra_history.json`), 기수통산성적비교 API

- **분석 로그 완전 저장**(`data/analysis_log/`) — 배당 타임라인·전적점수·이상감지·결과 원시 보존
- 발주시간 자동 감지(배당판→타이머 자동 시작), 마감 후 일괄 결과 등록, 일본 중앙(JRA)/지방(NAR) 분리
- 실제 투자금액 입력 필드 + 누적 손익 집계, 모바일 반응형 CSS

### 수정된 버그
- 한국 PDF 분석 405 근본 해결(`fitz` import try/except 방어)
- 전적 파서 오탐(334행 오즈표) 방어 + 전적 0두가 기존 전적 덮어쓰기 방지
- 팝업 자동전송 토글 클릭 불가 버그

### 보완점
- 거리·코스·기수이력 세부 미수집(부진마 "거리 변경/기수 교체" 조건 보류)
- 이상감지 UI 3단계(🔴/🟡/🟢) 종합으로 단순화
