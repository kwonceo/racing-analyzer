- **분석 고도화 2종: 적중 근거 표시 + 실시간 이상감지 고도화**: **[1번] 적중 근거**(`_apply_result_learning` `hit_basis`) — 결과/복기 리포트에 경주별 `🎯 적중 근거` 블록: 전적점수·급락점수·**급락폭**·역배열(감지·입상)·**최종신뢰도** 칩 + `📌` 한줄 근거 요약("전적유력마 1번 적중 · 1+2 복승 급락(-62.5%) → 1·2번 입상 · 역배열 감지말 3번 입상"). record·review 저장(재조회 유지). **[2번] 실시간 이상감지 고도화**(`_advanced_anomaly`, `signalQuality.advanced`): ①**급락 속도**(급락폭/수집간격 분) — 분당 10%+ 🔴 강한 신호·5%+ 🟡. ②**연속 하락 vs 단발 반등** — 3회 연속 하락(총 8%+)이면 종합 신뢰도 **+20**, 단발 후 반등이면 **−15**(신뢰도 재클램프 0~100). ③**페이크 베팅 감지** — 급락 후 반등(15%↓후 15%↑) `⚠️ 페이크 베팅 의심`. ④**복승 환급률 이상** — 전체 복승 역수합 + 상위 3조합 점유율 90%+ 시 `🟠 자금집중` 경고(이변 시 고배당). 신호 리스트·🎯 신호품질 카드에 표시. 격리 테스트(급락속도 분당 21%·연속하락/반등·페이크·환급률·신뢰도 보정 + 적중근거 생성) 통과
- **역배열 감지 특별 표시**(`_inverse_arrangement`, 추천 상단 `🔄 역배열 감지!` 배너): 단승 유력마 순서 ≠ 복승/쌍승 배당 순서(=실질 유력마 변경 가능성)를 **4유형 통합** 감지. ①**배너 상단부** — 단승 1위(+배당)·복승 최저 조합(단승1위 포함=정상/빠짐=불일치)·쌍승 역전 방향(비정상)·"→ 실질 유력마가 바뀌었을 가능성". ②**4유형별 표시** — `🔄 쌍승역전`(단승 N번 유력이나 쌍승 M번 1착이 더 낮음)·`⚠️ 복승불일치`(단승 예상 A+B, 실제 최저 A+C → C번 주목)·`📊 배당압축`(상위 말 배당 비정상 근접)·`🔴 집중급락`(N번 전체 평균보다 10%+ 더 급락). ③**추천 반영** — `⭐ 역배열 감지말`을 추천 상단에 별도 블록(빨강 강조)으로 표시(배당 높아도 우선 노출) + `복승 역배열` 조합(감지말 낀 저배당순)을 일반 추천과 구분 표시. 앞서 만든 `_win_exacta_reversal`/`_quinella_mismatch`/`_excess_drop_analysis` 결과를 재사용(중복 계산 없음)해 신호조합 추천(`_signal_combo_bets`)에도 이미 반영됨. 격리 테스트(단승 9번→실질 2번, 복승 2+9/2+5) 통과
- **핵심 이상감지 수학 공식 5종**(`_win_exacta_reversal`/`_quinella_mismatch`/`_signal_confidence`): ①**쌍승 역전 감지 공식** — 단승(미수집 시 복승인기) 유력마 A vs 다른 말 B 방향 비교. 역전비율 = 쌍승(B→A)/쌍승(A→B). <0.95 🟡 역전 · <0.80 🔴 강한역전 · <0.60 🔴🔴 압도적 역전("🔄 역전감지: 단승 9번 유력이나 쌍승에서 2번 1착이 더 낮음 → 실질 1착 2번 가능성"). ②**복승 불일치 감지 공식** — 불일치점수 = 단승 예상 최저복승(1+2위 조합)/실제 최저복승. 1.2+ 🟡 · 1.5+ 🔴 · 2.0+ 🔴🔴("⚠️ 복승 불일치: 예상 A+B / 실제 A+C → C번 집중 자금 유입"). ③**초과급락 기준 수정** — 전체 평균 대비 10%+ 초과(또는 절대 10%+)는 무조건 집중신호(노이즈 아님, 앞 커밋 ABS_STRONG 연계). ④**종합 신뢰도 점수** — 초과급락40% + 쌍승역전35% + 복승불일치25% 합산(말별 0~100). 70+ 🔴 강력 · 40~69 🟡 참고 · 40 미만 노이즈. ⑤**추천 조합 자동 반영** — 신뢰도 강력(70+)·참고(40~69) 말을 신호조합 추천 최우선 순위로 반영(배당 높아도 신호 있으면 표시). 프론트 🎯 신호 품질 분석 카드에 종합신뢰도표·쌍승역전·복승불일치 추가. 격리 테스트(역전 0.5/불일치 3.0/신뢰도 합산/70+ 강력 트리거) 통과
- **일본경마 삼복승 수집 제거 + 추정 보험 유지(절충)**(확장 v2.1.9): ①**수집 제거**(안정성) — `collectTripleKeiba`(keiba fetch)·`collectTripleByTabs`(사설 탭클릭) 양 경로에서 삼복승(3連複) fetch/탭클릭 완전 제거(일본=복승+쌍승, 한국=복승). 불안정하던 축 클릭 수집을 없앰. 기존 삼복승 수집 함수는 코드로 보존(호출만 안 함). ②**추천은 추정배당으로 유지**(절충) — 삼복승 배당 미수집 시 `_triple_analyze`가 삼복승 추천을 `_trio_est`(구성 복승 3쌍 기하평균×2) 추정배당으로 유지하되 복승 중심으로 '보험(추정)' 소액(총 ≤18%)만 배분하고 남는 몫은 복승 메인으로(`estimated` 플래그, 프론트 '추정보험' 배지). 이변 고배당 보험·4착 near-miss·대규모급락 보험 학습 유지. 삼복승 실배당이 수집되면 기존 로직 그대로(추정 아님). 쌍승 역전은 참고로 계속 표시. ③**UI 정리** — 팝업 라벨·수집 상태·index 안내 문구를 복승·쌍승 중심으로 정리

- **한국경마 결과 매칭 버그 수정(형식 불일치)**(`_resolve_race_key`): 분석 로그 raceKey(`2026-07-05 서울 5경주`)와 결과 입력 raceKey(`서울 5`) 형식이 달라 "분석 경주 없음"이 뜨던 문제. ①**유연 매칭 리졸버** — 우선순위 ①완전 일치 ②경마장+경주번호 일치(오늘 날짜 후보 우선) ③경주번호만 일치(후보 유일할 때만). 날짜·"경주" 접미사 없어도 매칭. ②**`_area_num` 보강** — 날짜 먼저 제거(번호 오검출 방지) + `서울 5`(접미사 없음)도 번호 추출 + 한글·**일본어(가나·한자) 경마장명** 지원 + **부경=부산 별칭 정규화**(`_track_norm`). ③**결과학습 재구성** — triple_store(활성)가 churn으로 비어도 `_rec_from_history`가 odds_history 스냅샷에서 rec를 재구성해 분석 재현(결과 입력 정상 처리). ④**후보 소스 통합**(`_analyzed_race_keys`) — 활성 배당+odds_history+analysis_log 전체에서 매칭(한국·일본 중앙/지방 모든 경기). ⑤**일괄 등록도 유연화** — `_match_row_to_key`가 리졸버 사용(일본 포함). `record-result` 응답에 실제 매칭 key·`matchedFrom` 반환. 테스트: 서울 1~10 + 부산(부경) 1~7 + 오늘우선 + 완전일치 + 일본중앙 전부 매칭 + 재구성 결과학습 확인

- **마감 오판(자동수집 조기 중단) 수정**(확장 v2.1.10): 발주 2분 40초 남았는데 "배당 5회 무변동"만으로 자동수집을 멈추던 문제. 확장이 발주시각을 못 잡으면(`deadline` 미설정) `!deadline`→무조건 무변동 마감을 적용하던 것을 제거. 배당판의 **실제 "남은시간"을 직접 파싱**(`pageRemainingMs`)해 **진짜 마감 임박(≤90초)일 때만** 무변동 마감 적용(발주시각 미검출 대비). 시간 정보 전무 시 DOM 마감 문구에만 의존.
- **첫 수집 가짜 급락 수정**: 배당판을 처음에 못 가져오거나(부분 로드) 시장 형성 초기 고배당일 때, 2번째 수집과 비교해 -90%대 가짜 급락이 이상감지 누적에 뜨던 문제. **첫 비교(수집 2건뿐)는 1틱 워밍업으로 급락 계산 보류**(`_triple_analyze` `market_forming`·`_history_append` 스냅샷 2건 이상부터 기록) → 2번째 수집을 기준으로 3번째부터 변동 계산.
- **경주 전환 시 배당 데이터 잔존 오검출 수정**: 새 경주로 넘어가도 이전 경주 배당이 남아 변동폭이 비정상적으로 크게 계산되던 문제(예: 이전 5+6=147.4 → 새 경주 5+6=5.7 → -96.1% 오검출) 해결. ①**경주 전환 감지**(`_baseline_reset_needed`) — 직전 대비 공통 복승 조합의 60%+가 90%+ 급락하면 다른 경주 잔존으로 판단, `triple_ingest`가 이전 history 완전 초기화 + 이번 수집을 새 기준값으로(`baselineReset`). ②**첫 수집 기준값** — 새 경주 첫 수집은 변동 계산 없이 기준값으로만 저장·"🎯 기준값 설정됨" 표시(`baselineSet`), 둘째 수집부터 변동 계산. ③**비정상 변동폭 필터** — 95%+ 급락 개별 조합은 복승/단승 drops에서 제외, 다수 발생 시 자동 재설정 + "⚠️ 비정상 변동폭 - 기준값 재설정" 신호. `_history_append`도 재설정 시 이상감지 생략·스냅샷 `baseline_reset` 표기. 서버단 방어라 확장 raceKey 갱신 지연/재사용에도 안전
- **현재 경주 자동 감지 오류 수정(확장 v2.1.7)**: 사설 보드에서 화면은 `고쿠라 4경주`인데 raceKey가 `모리오카 2경주`로 잘못 잡히던 문제. `extractRaceKey`가 경마장 목록(고쿠라 미포함)에서 본문의 첫 매칭(모리오카)을 집던 것을, **하단 네비 "이전 \<경마장\> \<N\>경주 … 다음"에 앵커링해 현재 경주를 직접 추출**하도록 수정(목록에 없는 경마장도 잡힘). 폴백 목록에 JRA 중앙(고쿠라·삿포로·니가타·도쿄·나카야마·주쿄·교토·한신) 추가. 고쿠라/부산/니가타 시나리오 검증.
- **한국경마 쌍승/삼복승 수집 완전 차단(확장 v2.1.6)**: 부산 등 KRA 경주인데도 `market`=auto + raceKey 추출 실패(사설 보드 표기 다양) 시 삼복승 탭을 클릭하던 문제. `pageLooksKorean()` 추가 — raceKey가 비어도 **페이지 본문/URL에 KRA 경마장명(서울/부산/제주/과천) + 경마 맥락**이 있으면 한국 확정. `isKoreaMode(raceKey, market)`로 판정 통일(종목=한국 OR raceKey OR 페이지). `collectTripleKeiba`·`collectTripleByTabs` 양 경로 적용 → 복승만 수집, 쌍승/삼복승 탭 클릭 완전 스킵. 로그 `[한국모드] 복승만 수집 - 쌍승/삼복승 생략`. 부산 화면 시나리오 포함 판정 검증.
